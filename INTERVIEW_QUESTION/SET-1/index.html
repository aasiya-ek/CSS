1.Flexbox:

Flexbox (Flexible Box Layout) is a CSS layout model that makes it easier to design and align items inside a container. It allows elements to automatically adjust their size, position, and spacing to fit different screen sizes and devices.

  1. To use Flexbox, we apply:- display:flex;
  2. The child elements inside it become flex items.
  3. Flexbox mainly controls:
    • Direction ‚Üí row or column (flex-direction)
    • Alignment ‚Üí horizontal & vertical (justify-content, align-items)
    • Flexibility ‚Üí grow, shrink, or stay fixed (flex-grow, flex-shrink, flex-basis)

Difference between flex row and column:

In Flexbox, the flex-direction property decides the main axis (the direction in which flex items are placed).
üîπ flex-direction: row (default)

  1. Items are placed horizontally (left to right).
  2. The main axis is horizontal.
  3. The cross axis is vertical.
  4. Example layout: [Item 1] [Item 2] [Item 3]

üîπ flex-direction: column

  1. Items are placed vertically (top to bottom).
  2. The main axis is vertical.
  3. The cross axis is horizontal.
  4. Example layout:
    [Item 1]
    [Item 2]
    [Item 3]

NOTE :
Row ‚Üí items go side by side.
Column ‚Üí items go one below the other.

2.Explaning inline, internal and external stylesheet

I. Inline Stylesheet:

Styles are written directly inside the HTML tag using the style attribute.
Affects only that element.

This is inline style

  • Use case: Quick testing or styling a single element.

  • Drawback: Hard to maintain when there are many styles.

II. Internal Stylesheet:

Styles are written inside the style tag in the HTML file‚Äôs head section.
Affects all elements in that page.

  • Use case: When you want styles for just one web page.

  • Drawback: Can‚Äôt reuse in multiple pages.

III. External Stylesheet:

Styles are written in a separate .css file, linked with link in the HTML head.
Affects all linked pages.

  • Use case: Best for large websites where the same CSS applies to many pages.

  • Advantage: Clean, reusable, easy to maintain.

NOTE :
Inline ‚Üí inside tag ‚Üí affects one element ‚Üí least preferred.
Internal ‚Üí style in HTML head ‚Üí affects that page only.
External ‚Üí separate CSS file ‚Üí reusable across pages ‚Üí most preferred.

3.Justify Content in Flexbox:

In Flexbox, the property justify-content controls how items are arranged along the main axis (the direction of flex-direction).
üîπ What justify-content does:
It decides the spacing and alignment of flex items inside the container horizontally (row) or vertically (column) depending on the flex direction.

Common Values of justify-content:

  1. flex-start (default) ‚Üí items stick to the start
  2. flex-end ‚Üí items stick to the end
  3. center ‚Üí items go to the center
  4. space-between ‚Üí equal space between items
  5. space-around ‚Üí equal space around items
  6. space-evenly ‚Üí equal space everywhere

4. Difference between Absolute and Relative Position:

üîπ position: absolute

  1. The element is positioned relative to the nearest positioned ancestor (ancestor with position: relative, absolute, or fixed).
  2. If no ancestor is positioned, it‚Äôs relative to the html /body (the page itself).
  3. The element is taken out of the normal flow, so it doesn‚Äôt leave space behind.

üîπ position: relative

  1. The element is positioned relative to its normal position.
  2. If we move it with top, left, right, or bottom, it shifts from where it would normally be, but the original space is still reserved.
  3. Other elements still behave like the box is in its original place.
üëâ In short:

Relative ‚Üí stays in flow, moves a little from its original spot.
Absolute ‚Üí leaves flow, sticks to ancestor or page.

5. Z-index

  1. z-index controls the stack order of elements (which one appears in front or behind).
  2. Higher z-index value ‚Üí element appears on top.
  3. Lower z-index value ‚Üí element goes behind.

Rules of Z-index :-

  • Works only on positioned elements (position: relative, absolute, fixed, sticky). If position: static (default), z-index won‚Äôt apply.
  • Default value is auto (means it follows normal stacking order).
  • Elements with the same z-index follow the HTML order (later one appears on top).
